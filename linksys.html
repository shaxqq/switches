sh interfaces status ethernet e”№” - посмотреть состояние порта

sh interfaces configuration ethernet e”№” - посмотреть состояние порта

sh interfaces description ethernet e”№” - посмотреть описание порта

sh bridge address-table ethernet e”№” - посмотреть MAC адрес

sh ip dhcp snooping binding ethernet e”№” - посмотреть привязки

sh ip source-guard status ethernet e”№” - посмотреть статус привязки

sh interfaces counters ethernet e”№” - посмотреть статистику об ошибках

test copper-port tdr e”№” - протестировать кабель

sh vlan — посмотреть Vlan на свиче

sh system – посмотреть информацию о свиче

sh cpu utilization – посмотреть загрузку ЦПУ свича


Прокинуть Vlan на порт
1. console – заходим в конфиг (config)
2. console(config)# – заходим на порт (interface ethernet e”№” )
3. console(config-if)# – switchport mode access
4. console(config-if)# – switchport access vlan ”№”

Прокинуть Vlan на 1ГБ порт
1. console – заходим в конфиг (config)
2. console(config)# – заходим на порт (interface ethernet g”№” )
3. console(config-if)# switchport trunk allowed vlan remove all
4. console(config-if)# – switchport mode access
5. console(config-if)# – switchport access vlan ”№”

Добавить/удалить Description
1. console – заходим в конфиг (config)
2. console(config)# – заходим на порт (interface ethernet e”№” )
3. console(config-if)# – description «описание»
4. console(config-if)# – no description – удалить описание

Вкл/Выкл Flowcontrol
1. console – заходим в конфиг (config)
2. console(config)# – заходим на порт (interface ethernet e”№” )
3. console(config-if)# – flowcontrol “on/off”

Вкл/Выкл отображение логов
1. console – заходим в конфиг (config)
2. console(config)# – no logging console

Если абоненты находятся в 1 свиче но не могут пропинговать друг друга.
1. console – заходим в конфиг (config)
2. console(config)# – interface vlan № (заходим во Vlan)
3. console(config-if)# - во влане пишем (no switchport protected-port fastethernet)
